As empresas vivem em um ambiente de rápida mudança, então os softwares tem de ser desenvolvido rapidamente para tirar proveito de oportunidades, com as empresas dispostas o sacrificar qualidade e requisito em prol de agilidade.E como as empresas operam em ambientes dinâmicos, é difícil definir todos os requisitos no começo, podendo ocorrer de so no final , com os usuários tendo a experiência que os requisitos aparecerem.


Processos de desenvolvimento com plano que especifica todos os requisitos, não são ágeis, partir do momento que surgem novos requisitos, necessitam de retrabalho, sendo ele um processo demorado que é entregue depois do prazo

 Para softwares críticos em que necessita de uma análise completa, a abordagem dirigida por plano é indicada.Mas para ambientes dinâmicos isso se torna problema, podendo ter mudanças radicais no final do processo.


A necessidade de desenvolvimento ágeis foi reconhecida há muito tempo, o desenvolvimento rápido decolou no final dos anos 90, quando surgiram os métodos ágeis. Os métodos ágeis se baseiam no desenvolvimento incremental de novas versões disponibilizadas para cliente a cada 2 ou 3 semanas, para obter feedback, além dos métodos diminuírem a documentação .
